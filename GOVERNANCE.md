# Governance

## Project Status

Tribunus is a **pre-alpha** project under active development. This governance model reflects that stage: lightweight, maintainer-directed, and designed to enable rapid iteration.

## Maintainer

Tribunus has a single designated **core maintainer**:

- **Julian Torres** — core maintainer and project lead

The core maintainer holds final decision authority on all matters including:

- Architecture and technical direction
- Product roadmap and prioritization
- Contribution acceptance and rejection
- Licensing and distribution terms
- Project policies and processes

## Maintainer Responsibilities

The core maintainer is responsible for:

- Keeping the project healthy, buildable, and focused
- Reviewing and responding to contributions in a reasonable timeframe
- Ensuring security-sensitive changes receive appropriate review
- Honoring the project's open-source license
- Maintaining public-facing documentation
- Communicating significant decisions to the community

## Contribution Model

Contributions follow an **issue-first** process:

1. Open or comment on an issue describing the proposed change
2. Reach alignment with the maintainer on approach before writing code
3. Submit a pull request referencing the issue
4. Maintainer reviews and either merges or provides feedback

Small, focused pull requests are strongly preferred over large monolithic changes.

## Community Governance

Tribunus does **not** have:

- A steering committee
- Formal voting procedures
- Elected or rotating leadership positions
- Any governance body beyond the core maintainer

This model will be revisited as the project matures.

## Code of Conduct

All participants — contributors, users, and commenters — are expected to:

- Be respectful and constructive in all interactions
- Provide honest, good-faith feedback and contributions
- Accept and offer critique professionally

Unacceptable behavior includes but is not limited to:

- Harassment, discrimination, or personal attacks
- Spam, trolling, or intentionally disruptive activity
- Bad-faith contributions or knowingly submitting harmful code

Violations should be reported to the core maintainer. Action is at the maintainer's discretion.

## Licensing

Tribunus is released under the **AGPLv3** license. Commercial dual-licensing is available for parties who require proprietary use. All licensing decisions — including commercial terms, exceptions, and license scope — are controlled by the core maintainer.

## Dispute Resolution

The core maintainer's decision on any project matter is final. If you disagree with the direction or decisions of the project, forking is always an option — the AGPLv3 license guarantees that right.
